# Share Feature — Design

**Date:** 2026-07-16
**Status:** Approved (via visual companion brainstorming)

## Goal
Let readers share the book on major tech/social platforms, with rich link
previews. Sharing is **page-contextual**: a chapter page shares that chapter; the
home page shares the book.

## Decisions (locked)
- **Two share scopes, two placements** (explicitly labelled so scope is never
  ambiguous):
  - **Top bar → "Share this book"** — always shares the book's home URL,
    regardless of which page you're on. The masthead represents the whole book.
    Injected into `.topbar__actions` on every page (popover menu).
  - **End of chapter → "Share this chapter"** — a labelled row of platform
    buttons inserted before the prev/next pager, on chapter pages only. Shares
    that chapter's canonical URL. Catches the reader at peak chapter-share intent.
  - Both reuse one link-builder (`HANDBOOK.shareItems`) and the same platform set.
  - Home / glossary / references show only the top-bar book share.
- **Platforms (in menu order):**
  - X, LinkedIn, Hacker News, Reddit, Bluesky
  - WhatsApp, Telegram
  - Copy link
  - "More…" → native OS share sheet via `navigator.share`, **shown only when the
    API exists** (i.e. mobile).
- **What is shared:** a **canonical deployed URL**, built from a fixed base
  (`https://sudhanshushekhar10.github.io/llm-handbook/`) + path — never
  `location.href` (so sharing works even when read locally). Top bar → book home;
  chapter block → that chapter.
- **Caption:** chapter block uses `"{chapter title} — from The Engineer's Field
  Guide to LLMs"`; top-bar book share uses the book title/tagline.
- **Open Graph / Twitter Card metadata:** added statically to every page's
  `<head>` (21 files) so LinkedIn/Slack/X/iMessage render a titled preview card.
  Crawlers don't run JS, so these tags cannot be JS-injected.
- **Share image:** one branded default 1200×630 PNG at
  `assets/og/og-default.png`, referenced by every page's `og:image`.
- **No trackers / no SDKs / no external scripts** — pure share-intent URLs,
  consistent with the site's "runs from file://" ethos.

## Components
1. **`nav.js`**
   - `HANDBOOK.meta.canonical` — the deployed base URL.
   - `HANDBOOK.canonicalUrl(slug)` — slug → absolute canonical URL
     (home → base; glossary/references → base+file; chapter → base+chapters/slug).
   - `HANDBOOK.shareMeta(slug)` — `{ url, title, caption }` from the registry.
   - `HANDBOOK.renderShare(slug)` — inject the Share button + popover into
     `.topbar__actions`; wire platform links, Copy link (clipboard + feedback),
     and the mobile native-share entry.
2. **`main.js`** — call `H.renderShare(PAGE.slug)` on boot.
3. **`styles.css`** — `.sharebtn`, `.sharemenu`, `.sharemenu__item` styles,
   theme-aware, matching the existing `.iconbtn`/modal look. Popover closes on
   outside-click and Esc.
4. **OG image** — generated by rendering an on-brand HTML card at 1200×630 and
   screenshotting it (Playwright), saved as `assets/og/og-default.png`.
5. **OG tags** — a one-shot script inserts an idempotent `og:`/`twitter:` block
   into each page's `<head>`, using per-page title/description from the registry.
   Re-runnable (skips files that already have the block).

## Accessibility / behavior
- Share button: `aria-haspopup`, `aria-expanded`, `aria-label="Share"`.
- Popover: keyboard-closable (Esc), click-outside to dismiss, focus returns to
  button.
- Platform links open in a new tab (`target="_blank" rel="noopener"`); "Copy
  link" and native share act in place.

## Out of scope (future)
- Per-chapter share images.
- Share counts / analytics.

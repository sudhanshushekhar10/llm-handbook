How to read this guide
01 · ORIENTATION

Start with analogies

Every concept is mapped to something you already know — a lexer, a cache, a git patch, a load balancer. The ML word comes second.

02 · DEPTH ON DEMAND

Skim, then dive

Read the narrative top-to-bottom; open the deep-dive panels only when you want the math. Hover any dotted term for a definition.

03 · HANDS-ON

Run the code

Tutorials ship with a Local (Mac/MPS) tab and a Cloud (CUDA) tab. Copy, run, break, learn.

04 · RECEIPTS

Follow the citations

Numbered chips like link to the original paper. The field is young enough to read the sources.
